一、 项目定位与核心价值
本项旨在基于火山引擎Coze扣子平台，打造一个专属于南京大学学生的AI智能体助手。通过整合南大官方教务、考研保研、校园公众号等多源数据，解决目前校园信息碎片化、检索难、时效慢的痛点，为南大学子提供一站式的学术、生活、生涯规划的AI学长式服务。

二、 核心功能模块详细扩充
问答互动拓展功能（智能检索与学术辅助）
多源融合检索（RAG - 检索增强生成）
深度政策解读：用户提问“今年XX学院保研需要什么条件？”或“某课程期末挂科了怎么补考？”，Agent优先从校内政策知识库中检索，结合大模型输出条理清晰的步骤。
实时动态追踪：若用户询问最新通知，Agent触发联网搜索，抓取最新南大官方公告。
出处透明化：所有基于知识库或网页生成的回答，均需在末尾或对应段落标注“数据来源：南京大学本科生院官网 / 2026版XX教材”，并提供原文超链接或文本切片，拒绝AI瞎编。
思维导图与复习框架一键生成
教材结构化梳理：输入某门高难课程（如《数据结构》、《数字电路》）的章节或核心概念，Agent会自动输出Markdown格式的阶梯大纲，或直接生成Mermaid逻辑代码。用户可直接复制到Notion、XMind中秒变图形化思维导图。
自主聊天与个性化（用户肖像与校园融入）
多维动态话题发起（主动关怀机制）
时序与关键节点触发：Agent不仅被动回答，还会利用Coze的定时触发器或根据日期主动提示。例如：
期末周前夕：“马上要进入期末周了，需要我帮你整理一下《计算系统基础》的历年常考知识点和思维导图吗？”
保研夏令营/预推免期（9月）：“学弟/学妹，XX学院的推免申请系统今天开放了，你要不要先看一下我为你整理的材料准备清单？”
情感陪伴：在用户面对学业压力吐槽时，提供有温度的、符合南大校园氛围的学长学姐式鼓励。
资料库的辅助读写功能（类Notebook个人看板）
多级知识库架构（公共库 + 个人库）
公共知识库（增/删/改）：由开发团队（你们小组）维护，涵盖南大官方教材、保研政策、考研历年真题。支持运营端一键批量增删。
个人专属知识库（Notebook）：每个用户拥有独立的加密存储空间。
一键“划重点”与动态错题本

三、 详细技术路径与Coze平台落地方案
为了在火山引擎Coze平台上100%落地上述功能，小组需要按照以下四个层级进行开发配置：
数据层（Data Layer）：多源多模态知识库构建
解决信息输入、反爬与定时更新问题：
静态/半静态数据（教材、保研历史指南）：
将PDF、Word、Markdown格式的复习资料进行人工清洗（去除错别字和排版错乱），直接上传至Coze的Knowledge（知识库）。
动态数据A（南大各官方网站）：
在Coze知识库中选择“Web URL”类型，直接输入南京大学本科生院、研究生院、各二级学院官网的通知公告栏URL。
定时更新设置：在Coze界面配置自动同步策略（按天/按周自动刷新），Coze的后台爬虫会自动重新抓取该网页并更新切片，实现免代码维护。
动态数据B（微信公众号 - 核心攻关）：
由于微信反爬极严，直接放链接会失效。
工程平替方案：利用开源工具（如RSSHub）将“南大招生小蓝鲸”等核心公众号转化为标准的RSS订阅源。在Coze中配置webhook或调用第三方RSS插件，定期抓取文本并利用Coze API动态写入知识库。
逻辑层（Logic Layer）：问答路由工作流（Workflows）
使用Coze的图形化工作流搭建复杂的思考路径：
意图识别节点（LLM）：判断用户输入的是“日常闲聊”、“要导图”、“查南大政策”还是“记录笔记”。
条件分支（Condition）：
分支一（查政策/教材）：触发Knowledge检索节点 -> 提取出处 -> 大模型整合润色 -> 输出。
分支二（要最新新闻）：若知识库未命中，触发Google/Bing Search联网插件 -> 抓取快照 -> 总结输出。
分支三（记笔记）：触发Database写入节点，将文本和时间戳存入数据库。
存储与记忆层（Storage Layer）：内置数据库（Database）
实现类Notebook的记忆与肖像功能
在Coze的Database中建立两张核心表：
user_profile（用户画像表）：字段包括user_id, grade（年级）, major（专业）, intent（保研/考研/出国倾向）。
user_notebook（个人笔记表）：字段包括note_id, user_id, category（错题/考点/日程）, content（内容）, create_time。
在Bot对话中，AI随时可以通过SQL节点或内置的Data操作节点读取这些数据，确保每次对话都记得用户的专业和过去的笔记。
表现层（Agent Layer）：人设（Persona）提示词编写
在Coze的Bot系统提示词中，注入精准的Prompt控制AI的语气和安全边界：
角色设定：
你是由南京大学AI创新实践小组开发的“小蓝鲸校园助手”。你是一位正在仙林/鼓楼校区就读、热心博学、幽默风趣的南大资深学长/学姐。
核心技能：
当用户询问南大教务、保研、考研、课程等学术问题时，必须强制检索南大知识库或调用联网搜索。回答时须严格遵守[来源: XXX]的格式注明出处。
绝对不允许编造不存在的学校政策或教务规定。若知识库与网络均未查到，请礼貌引导用户：“这个规定学长也没在官网上查到，稳妥起见建议你发邮件问问院里的教务老师或者辅导员哦~”
